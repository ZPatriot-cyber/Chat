# 💬 TCP Chat — C# / WPF

> Многопользовательский чат на TCP-сокетах с WPF-интерфейсом

---



## Быстрый старт

### 1 — Запустить сервер

```bash
cd ChatServer1
dotnet run
```

Сервер поднимется на **порту 8888**.

### 2 — Запустить клиент

Откройте `ChatClient1` в Visual Studio и нажмите **F5**, или запустите `ChatClient1.exe` напрямую.

В окне подключения введите:

| Поле | Пример |
|------|--------|
| IP-адрес | `127.0.0.1` |
| Порт | `8888` |
| Никнейм | `Alice` |

Нажмите **«Подключиться»** — и вы в чате.

---

## Команды

| Команда | Пример | Описание |
|---------|--------|----------|
| *(просто текст)* | `Всем привет!` | Сообщение всем |
| `/pm <ник> <текст>` | `/pm Bob Привет!` | Личное сообщение |
| `/users` | `/users` | Список онлайн-пользователей |

> ⚠️ Никнейм не должен содержать пробелы.

## Оформление сообщений

| Тип | Выравнивание | Фон |
|-----|-------------|-----|
| Моё сообщение | Справа | 🟢 светло-зелёный |
| Чужое сообщение | Слева | ⬜ белый |
| Личное (входящее) | Слева | 🔵 голубой |
| Личное (исходящее) | Справа | 🟣 сиреневый |
| Системное | Слева | 🟡 жёлтый |

## Архитектура

### Сервер `ChatServer`

```
TcpListener
    └── ClientHandler (отдельный поток на каждого клиента)
            ├── BroadcastMessage()     — сообщение всем
            ├── SendPrivateMessage()   — личное сообщение
            └── UpdateUserList()       — обновить список пользователей

Dictionary<string, ClientHandler>     — реестр подключённых клиентов
```

События: `OnMessageReceived` · `OnClientConnected` · `OnClientDisconnected`

### Клиент `ChatClient`

```
TcpClient
    └── NetworkStream
            └── фоновый поток чтения

События: OnConnected · OnMessageReceived · OnError
```

### WPF `MainWindow`

```
ObservableCollection<ChatMessage>     ← привязка к списку сообщений
INotifyPropertyChanged                ← авто-обновление UI
Dispatcher.Invoke                     ← потокобезопасность
```

---

## Известные ограничения

- **1 поток = 1 клиент** — при 100+ подключениях возможны проблемы с производительностью
- История сообщений хранится только **в памяти** (нет персистентности)
- Отсутствует **шифрование** трафика
- Нет таймаутов для неактивных клиентов
